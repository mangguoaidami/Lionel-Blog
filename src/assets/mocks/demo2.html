What will we do:

  1. Compare Unit/Component with E2E Testing.
  2. Create or use a@angular/cli based project.
  3. Adapt the e2e page object to match our app.
  4. Write e2e tests that walk through a process.
  5. Run the tests

1.Difference Unit/Component vs. End 2 end Testing:

Unit/Component:

  1. Test a single Component, Service , Pipe etc.
  2. Test a single specific behavior in a controlled environment.
  3. Mock everything you need to test this functionality.
  4. Use code coverage analysis to make sure everything is covered.
  5. Improves code structure and quality.
  6. Test edge cases on the most detailed level.
  7. Know if a change breaks a specific behaviour.
  8. Dose not test if the whole application or a process works unreal life.

End to end Testing

  1. Test complete processes “End 2 End”.
  2. Test in real environments with the whole application.
  3. Test real live situations.
  4. Know if most important features work with the complete application
  5. Focussed on the overall functionality and not the specific code structure.
  6. Does not test Edge cases.
  7. Does not necessarily improve code quality.

Conclusion: You need both e2e and unit/component testing!
